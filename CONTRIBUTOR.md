# 기여를 환영합니다.
- 문제 리뷰 혹은 문제집 생성 `이슈 생성`
- Study Note 공유 `library 폴더`

## 리뷰 참여 방법
**사전 준비**
- 이 저장소를 `Fork`를 하여 문제 풀이 개인저장소를 준비해주세요.
- 문제 풀이는 `users/{nickname}` 폴더를 생성후 참여하시면 됩니다.

### Issue 생성 절차(문제집 생성)
1. 함께 풀고싶은 문제들에 대해서 링크를 포함한 문제집을(`Issue`) 만듭니다. ([ex: #4](https://github.com/PS-ATTACK/PS-ATTACK/issues/4))
2. (필수) `난이도 라벨`을 첨부해주세요.

### 문제풀이 절차
1. `Open된 Issue` 문제를 개인 저장소에서 풀어주세요.
2. 해당 `Issue`를 `Tagging`하여 `PR`을 요청하면 완료됩니다.

> 만약 `PR` 요청이 아닌 문제 풀이 링크가 있는 `Issue`를 요청한다면 `해당 Issue`에 `Comment`를 통해서 개인 리포지토리 `Issue` 링크를 멘션해주세요.   
> (필수) 문제 풀이 파일로 갈수있는 링크를 제공해야합니다.

### 문제리뷰 절차
1. 링크된 문제 풀이(`PR` 혹은 개인링크)에 대해서 궁금하거나, 개선점을 `Comment`로 남겨주세요.
2. (선택) 추천할만한 `풀이`에 공감을 해주세요.

### 리뷰를 위한 좋은 코드 
- 풀었던 문제 소스 상단에 `문제 링크`와 `접근 방법` 및 `시/공간 복잡도`등 문제 이해에 대한 정보를 작성해주면 감사합니다.
- 짧은 코드를 우선하지 않고 잘 읽히는 코드를 선호합니다.
